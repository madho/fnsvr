<task type="auto" tdd="true">
  <name>Task 1: Create scheduler.py with dynamic plist generation and launchctl wrappers</name>
  <files>src/fnsvr/scheduler.py, tests/test_scheduler.py</files>
  <behavior>
    - test_scan_plist_contains_correct_interval: generated scan plist XML has StartInterval 14400 and RunAtLoad true
    - test_digest_plist_contains_calendar_interval: generated digest plist XML has StartCalendarInterval with Weekday=1, Hour=8, Minute=0
    - test_plist_has_absolute_binary_path: plist ProgramArguments[0] is an absolute path (starts with /)
    - test_plist_has_log_paths: plist StandardOutPath and StandardErrorPath point to ~/.fnsvr/data/logs/
    - test_plist_label_correct: scan plist Label is com.fnsvr.scan, digest is com.fnsvr.digest
    - test_find_fnsvr_binary_returns_absolute_path: _find_fnsvr_binary returns an absolute path string
  </behavior>
  <action>
    Create src/fnsvr/scheduler.py with these functions:

    1. _find_fnsvr_binary() -> str: Detect the absolute path to the fnsvr executable. Strategy: first try shutil.which("fnsvr"), if that returns None fall back to Path(sys.executable).parent / "fnsvr", if that does not exist fall back to f"{sys.executable} -m fnsvr" (list form for ProgramArguments). Return the absolute path string.

    2. generate_scan_plist(binary_path: str, log_dir: Path) -> str: Return XML string for com.fnsvr.scan.plist. ProgramArguments = [binary_path, "scan"]. StartInterval = 14400 (4 hours). RunAtLoad = true. WorkingDirectory = str(Path.home()). StandardOutPath = str(log_dir / "scan.log"). StandardErrorPath = str(log_dir / "scan-error.log"). Label = "com.fnsvr.scan".

    3. generate_digest_plist(binary_path: str, log_dir: Path) -> str: Return XML string for com.fnsvr.digest.plist. ProgramArguments = [binary_path, "digest"]. StartCalendarInterval with Weekday=1, Hour=8, Minute=0. StandardOutPath = str(log_dir / "digest.log"). StandardErrorPath = str(log_dir / "digest-error.log"). Label = "com.fnsvr.digest".

    4. install_schedule(log_dir: Path | None = None) -> tuple[Path, Path]: Find binary, generate both plists, write to ~/Library/LaunchAgents/, create log_dir (default ~/.fnsvr/data/logs/), run "launchctl load {plist_path}" via subprocess.run for each. Return tuple of plist paths. Raise RuntimeError if not macOS (platform.system() != "Darwin").

    5. uninstall_schedule() -> tuple[bool, bool]: For each plist in ~/Library/LaunchAgents/ (com.fnsvr.scan.plist, com.fnsvr.digest.plist), run "launchctl unload {path}" then delete the file. Return (scan_removed, digest_removed). Do not error if files do not exist.

    6. schedule_status() -> dict: Check if each plist exists in ~/Library/LaunchAgents/. For each existing plist, read it and extract Label. Return dict with keys "scan" and "digest", each having "installed" (bool) and "plist_path" (str | None).

    Use plistlib from stdlib to generate plist XML (not string concatenation). Per SCHED-03, all paths must be absolute -- no relative paths or shell expansion in the plist.

    Create tests/test_scheduler.py with the behavior tests above. Tests should call generate_scan_plist and generate_digest_plist with known paths and parse the returned XML with plistlib.loads() to verify structure. Do NOT test install/uninstall (those call launchctl). Mock _find_fnsvr_binary where needed.
  </action>
