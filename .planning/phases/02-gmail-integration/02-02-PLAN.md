Create src/fnsvr/downloader.py with these exact functions:

**Imports:**
```python
from __future__ import annotations
import base64
import logging
import os
import re
import sqlite3
from pathlib import Path
```

Use `logger = logging.getLogger(__name__)` for all logging.

**sanitize_filename(filename: str) -> str (ATT-03):**
- Replace any character that is NOT alphanumeric, dot, underscore, or hyphen with underscore
- Use: `re.sub(r"[^a-zA-Z0-9._-]", "_", filename)`
- If result is empty after sanitization, return "unnamed_attachment"

**unique_path(path: Path) -> Path (ATT-04):**
- If path does not exist, return it unchanged
- Otherwise, split into stem and suffix
- Try stem_1.suffix, stem_2.suffix, etc. until a non-existing path is found
- Return the first non-existing candidate
- This guarantees no file is ever overwritten

**walk_parts(parts: list[dict] | None) -> list[dict]:**
- If parts is None or empty, return []
- For each part: if it has nested "parts", recurse into them
- Otherwise, append the leaf part to results
- Return flat list of all leaf parts (handles nested multipart/mixed MIME structures)

**download_attachment(service, message_id: str, attachment_id: str, filename: str, save_dir: Path) -> tuple[str, int]:**
- Call `service.users().messages().attachments().get(userId="me", messageId=message_id, id=attachment_id).execute()`
- Decode data: `base64.urlsafe_b64decode(response["data"])` (Gmail uses URL-safe base64)
- Sanitize filename with sanitize_filename()
- Compute save path: `unique_path(save_dir / safe_name)`
- Write bytes: `save_path.write_bytes(data)`
- Return `(str(save_path), len(data))`

**process_attachments(service, conn: sqlite3.Connection, email_row_id: int, message_id: str, parts: list[dict], save_dir: Path, allowed_extensions: list[str]) -> int (ATT-01, ATT-02, ATT-03, ATT-05):**
- Call walk_parts(parts) to flatten nested MIME structure
- For each leaf part:
  - Get filename from part.get("filename", "")
  - Skip if no filename or empty filename
  - Get file extension: `Path(filename).suffix.lower()`
  - Skip if extension not in allowed_extensions (per ATT-02)
  - Get mime_type from part.get("mimeType", "")
  - Determine attachment data source:
    - If part["body"].get("data") exists, decode inline: `base64.urlsafe_b64decode(part["body"]["data"])`
    - Elif part["body"].get("attachmentId") exists, call download_attachment(service, message_id, attachment_id, filename, save_dir)
    - Else: skip this part (no data available)
  - For inline data path: sanitize filename, compute unique_path, write_bytes
  - Insert into attachments table:
    ```python
    conn.execute(
        "INSERT INTO attachments (email_id, filename, local_path, mime_type, size_bytes, downloaded) VALUES (?, ?, ?, ?, ?, ?)",
        (email_row_id, filename, local_path, mime_type, size_bytes, 1)
    )
    conn.commit()
    ```
  - Increment download_count
  - Wrap EACH attachment in try/except (per ATT-05):
    - On failure: log error with `logger.error(f"Failed to download {filename} from message {message_id}: {exc}")`
    - Record failure in DB: INSERT with downloaded=0, local_path=None
    - Continue to next attachment (do NOT raise)
- Return download_count

**save_dir construction (ATT-03):**
The caller (scanner.py in Plan 03) will construct save_dir as:
`config.resolve_path(config["paths"]["attachments"]) / account["name"]`
and pass it to process_attachments. The downloader itself does NOT read config -- it receives save_dir as a parameter.

Ensure save_dir exists: `save_dir.mkdir(parents=True, exist_ok=True)` at the start of process_attachments.

Add type hints and docstrings to all public functions. Follow existing code style from config.py/storage.py.
