Create src/fnsvr/reviewer.py with module docstring: "Interactive review workflow for fnsvr."

Import: `from __future__ import annotations`, `import sqlite3`, `import logging`, `from fnsvr import storage`.

Functions:

1. `format_email(email, index: int, total: int) -> str`:
   - Returns a formatted string displaying one email for review.
   - Format (per REV-02):
     ```
     --- [{index}/{total}] ---
     Priority:    {priority}
     Category:    {category}
     Subject:     {subject}
     From:        {sender}
     Date:        {date_received}
     Account:     {account_name}
     Attachments: {"Yes" if has_attachments else "No"}

     {snippet if snippet else "(no preview)"}
     ```
   - Access fields via email["field_name"] (sqlite3.Row supports dict-like access).

2. `review_interactive(conn: sqlite3.Connection, emails: list) -> int`:
   - Takes a list of email Row objects (already filtered by caller via get_unreviewed).
   - If `len(emails) == 0`, print "No unreviewed items." and return 0.
   - Print f"Found {len(emails)} unreviewed item(s).\n"
   - Loop through emails with index tracking. For each email:
     a. Print `format_email(email, i+1, len(emails))`
     b. Print prompt: `"[y] mark reviewed  [n] skip  [q] quit  [a] mark all remaining > "`
     c. Read input via `input()`. Strip and lowercase.
     d. If "y": prompt for optional notes via `input("Notes (Enter to skip): ")`. If notes is empty string, pass None. Call `storage.mark_reviewed(conn, email["id"], notes)`. Increment reviewed_count.
     e. If "n": continue to next item.
     f. If "q": break out of loop.
     g. If "a": call `mark_all(conn, emails[i:])` (current item onwards). Add len(emails[i:]) to reviewed_count. Break. (per REV-03)
     h. Any other input: print "Unknown option. Try y/n/q/a." and re-prompt same item (use while loop inside for loop).
   - Print f"\nReviewed {reviewed_count} item(s)."
   - Return reviewed_count.
   - Wrap the entire interactive loop in try/except for KeyboardInterrupt and EOFError, printing "\nReview cancelled." and returning the count so far.

3. `mark_all(conn: sqlite3.Connection, emails: list, notes: str = "Bulk reviewed") -> int`:
   - Iterate emails, call `storage.mark_reviewed(conn, email["id"], notes)` for each.
   - Return count of items marked (per REV-05).
   - Used both by interactive "a" command and by CLI --mark-all flag.

Per REV-06: Review status and notes are persisted via storage.mark_reviewed which writes to SQLite. The reviewed column is already in the schema with a timestamp via the existing mark_reviewed function.

Per REV-04: Filtering by --category and --account is handled at the CLI layer (Plan 04) by passing those args to storage.get_unreviewed. reviewer.py receives the pre-filtered list.
