## Design System

**Colors (CSS custom properties on :root):**
- --bg: #0a0a0a (near-black background)
- --bg-elevated: #141414 (cards, code blocks)
- --bg-code: #1a1a1a (terminal blocks)
- --text: #d4d0c8 (warm off-white, not pure white)
- --text-muted: #8a8680 (secondary text)
- --accent: #e8a040 (warm amber -- the signature color)
- --accent-dim: #c4863a (hover/secondary amber)
- --border: #2a2a2a (subtle dividers)
- --critical: #e85040 (for critical priority badges)
- --high: #e8a040 (for high priority badges, same as accent)

**Typography (system fonts only, zero external requests):**
- --font-mono: "SF Mono", "Cascadia Code", "JetBrains Mono", "Fira Code", Menlo, Monaco, Consolas, monospace
- --font-body: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif
- Product name "fnsvr" always rendered in monospace and amber
- Headings in body font, bold, large
- Body text 17-18px for readability, line-height 1.6

**Layout:**
- Single column, max-width 720px, centered
- Generous vertical padding between sections (80-120px)
- No sidebar, no multi-column grid except the feature grid
- Mobile-first: works on 320px+ screens

**Motion:**
- Respect prefers-reduced-motion (disable all animation when set)
- Subtle fade-in-up on scroll for sections using IntersectionObserver (vanilla JS, ~15 lines)
- No other animation

## Page Structure (7 sections, in this exact order)

### 1. Hero
- Small monospace label at top: "fnsvr" in amber
- Large headline (h1): "Your K-1 is buried in your third inbox." (or similar -- the point is naming a specific, painful, relatable financial email problem)
- Subheadline (p): One sentence describing what fnsvr is. Example: "A local CLI tool that scans your Gmail accounts for financial emails you can't afford to miss."
- Single CTA button: "View on GitHub" linking to https://github.com/madho/fnsvr -- styled as a bordered button with amber text, not a filled button. Opens in new tab.
- Generous whitespace above and below. No illustration, no screenshot, no icon.

### 2. The Problem
- Section heading: "The problem" (h2)
- 2-3 short paragraphs drawn from Docs/VISION.md "The Problem" section. Adapt the language to be direct and second-person ("You have 3+ inboxes..."). Key points:
  - Financial emails get buried across multiple inboxes
  - Unlike a missed meeting, a missed K-1 or unsigned equity doc has real dollar consequences
  - Gmail filters don't work across accounts, can't prioritize by financial urgency, can't create an audit trail
- Keep it tight. No more than 100 words total.

### 3. What fnsvr Detects (5-category feature grid)
- Section heading: "What it catches" (h2)
- CSS Grid: 3 columns on desktop (with last 2 items centered), 1 column on mobile
- Each card: dark elevated background (--bg-elevated), subtle border, rounded corners
- Each card contains:
  - Category label in amber (from config.example.yaml)
  - Priority badge: "critical" or "high" -- small pill, red for critical, amber for high
  - 2-3 example patterns as a compact list in muted text

The 5 categories and their content (pulled from config.example.yaml):

1. **Tax Documents** -- priority: critical
   Examples: K-1s, 1099s, W-2s, tax statements

2. **Signature Requests** -- priority: critical
   Examples: DocuSign, HelloSign, PandaDoc, equity agreements

3. **Equity & Options** -- priority: critical
   Examples: RSU vesting, stock options, 409A, ESPP, cap table updates

4. **Brokerage Statements** -- priority: high
   Examples: Account statements, trade confirmations, dividends, margin calls

5. **Bank & Wire Transfers** -- priority: high
   Examples: Wire confirmations, ACH transfers, balance alerts, large transactions

### 4. How It Works (3 steps)
- Section heading: "How it works" (h2)
- Three numbered steps, each with a monospace step number and short description:

  1. **Install** -- `pip install fnsvr` or `brew install fnsvr`
  2. **Scan** -- `fnsvr scan` checks all your Gmail accounts (read-only, local-only)
  3. **Review** -- Notifications, digests, and an interactive terminal review queue

- Display each step as a horizontal row on desktop (3 columns), vertical stack on mobile
- Step numbers in large amber monospace
- Keep descriptions to one line each

### 5. Design Principles
- Section heading: "How it's built" (h2)
- Four principles, each as a short bold statement followed by one sentence of explanation. Pull text from Docs/VISION.md "Design Principles" section, but tighten and rephrase:

  1. **Local-first, always.** Your email data never leaves your machine. No cloud, no accounts, no SaaS.
  2. **Read-only by design.** The Gmail scope is gmail.readonly. Architecturally incapable of touching your inbox.
  3. **Config over code.** Add a new bank or broker with a one-line YAML edit. No code changes needed.
  4. **Open source.** MIT licensed. Audit the code, extend the patterns, own the tool.

- Use a simple vertical list with generous spacing. No cards needed here.

### 6. Quick Start (terminal block)
- Section heading: "Get started" (h2)
- A styled terminal/code block with dark background (--bg-code), amber prompt characters, and monospace text
- Content:

```
$ pip install fnsvr
$ fnsvr init
$ fnsvr setup personal
$ fnsvr scan --initial
```

- Below the block: one line of muted text: "Scans the last 90 days across all configured accounts."
- Optional: minimal typing animation for the terminal block using vanilla JS IntersectionObserver (types out lines one by one when scrolled into view). Respect prefers-reduced-motion -- if set, show all lines immediately with no animation.

### 7. Footer
- Simple, minimal. One line layout.
- Left: "fnsvr" in monospace amber
- Center or right: "GitHub" link (to repo), separator dot, "MIT License"
- Below: "Built by madho" in muted text
- No newsletter signup, no email capture, no social links, no "back to top"

## Voice and Tone

- Direct, confident, zero fluff
- No "supercharge", "never miss another", "powered by AI", "10x your productivity"
- Talk to the reader like a peer, not a customer
- State facts about what the tool does. Let the reader decide.
- Slightly dry. The tool is serious because the problem is serious, but the copy should not be dramatic.

## What to Avoid

- Fintech blue or purple gradients
- Dashboard screenshots or mockups
- Testimonial sections, pricing tables, "trusted by" logos
- Cookie banners (there are no cookies to consent to)
- Hamburger menus or complex navigation (single page, no nav needed)
- Any JS framework or library
